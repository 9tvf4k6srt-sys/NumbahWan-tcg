Merging Sections Increases Engagement 2026-02-04
Combining Absurd Wing and Deep Dives into "The Abyss" creates a single exploration destination with 16+ pages. Users don't have to choose between sections - they can seamlessly explore both lore AND physical locations in one journey. This increases time-on-site and discovery.
UX Navigation Engagement
Digital→Physical Bridge Creates Community 2026-02-04
The Tabletop Realm creates a bridge between digital cards and real-life play. This transforms online-only players into IRL friends. Key insight: the conversion formula (ATK/5=STR) makes any card playable without needing new assets. Reuse existing content in new formats.
Community Content Reuse IRL
Cloudflare Pages Routing Patterns 2026-02-04
Key learnings from today's routing fixes:
1. _routes.json has a 100-rule limit - use wildcards like /lore/* instead of individual files
2. Worker-handled routes need to explicitly serve HTML files
3. serveStatic must use 'hono/cloudflare-workers', NOT '@hono/node-server/serve-static'
4. Post-build scripts can modify _routes.json but must use .cjs extension for ES modules
Cloudflare Routing Debugging
nanobanana = Consistent Art Quality 2026-02-04
Using a single AI art model (nanobanana) for all hero images ensures visual consistency across the site. Mixed models = jarring experience. Generated 12 hero images today with consistent quality because of standardized prompts and single model usage.
Art Consistency AI Generation
Reusable Components = 10x Speed 2026-02-03
The Deep Dive template engine (nw-deepdive.js) allows creating new lore pages in minutes instead of hours. Define section types once, reuse forever. Same pattern applies to: navigation (nw-nav.js), wallet (nw-wallet.js), currency (nw-currency.js).
Architecture DRY Speed
Our Building Philosophy

Building efficiently is only half the equation. Continuous improvement on HOW we build is equally important. We don't just ship features—we obsess over the process that creates them.

Reusable Components

Every feature built should be a building block for future features. The Deep Dive Engine powers all lore pages. nw-nav.js works on every page. Build once, use everywhere.

Measure Everything

Track build times, bundle sizes, and development velocity. What gets measured gets improved. This dashboard exists because we measure our own efficiency.

Iterate Constantly

Don't wait for perfection. Ship, learn, improve. Each iteration makes the next one faster. The Abyss is v3 of our section organization.

Balance Build & Learn

60% building new features, 30% improving existing systems, 10% learning. This ratio keeps us moving forward while getting better.

Automate Repetition

If you do it twice, automate it. Art generation prompts, deployments, routing scripts—all streamlined. Never do manually what a script can do.

Document Patterns

Successful patterns become templates. Failures become lessons. Both improve future efficiency. This Learning Log is our institutional memory.

Quality is Non-Negotiable

Fast shipping doesn't mean low quality. Our design system ensures everything looks good. Reusable components ensure consistency. Speed AND quality.

Build for Community

Every feature should strengthen guild bonds. Tabletop Realm turns online friends into IRL adventurers. The Abyss gives explorers endless content to discuss.
